## Spacing Scale

All values are SwiftUI points (1pt = 1px on non-retina, scales on retina). Scale is inherited from Phase 1 and applies unchanged.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4pt | Icon gaps, inline label padding |
| sm | 8pt | Compact element spacing within cards, pill internal padding |
| md | 16pt | Default element spacing, panel horizontal padding |
| lg | 24pt | Section padding, between trip cards |
| xl | 32pt | Layout gaps, panel header spacing |
| 2xl | 48pt | Major section breaks |
| 3xl | 64pt | Page-level / full-bleed spacing |

Exceptions (Phase 3 additions):
- Touch targets (all buttons including "+" and Stop Trip): minimum 44pt hit area per Apple HIG
- Drag strip interaction height: 44pt tall touch area even if visual height is smaller
- Recording pill: minimum 44pt height to meet touch target for Stop Trip button
- Photo gallery thumbnails: 80Ă—80pt visual size with 4pt corner radius, 8pt gap between thumbnails
- Trip detail map: fixed 240pt height
- Place pin callout: no minimum â€” callout is system MKAnnotationView, not a custom tap target

Source: Phase 1 UI-SPEC spacing exceptions; CONTEXT.md D-15 (map height 240pt), D-16 (thumbnails 80Ă—80pt)

## Color

All colors use the established `Color.Nomad` namespace. No new color values are introduced in Phase 3.

| Role | Hex | SwiftUI Name | Usage |
|------|-----|-------------|-------|
| Globe Background | `#0A0A14` | `Nomad.globeBackground` | RealityKit scene background only |
| Dominant (60%) | `#FAF8F4` | `Nomad.cream` | ProfileSheet + TripDetailSheet backgrounds (via `.panelGradient()`) |
| Secondary (30%) | `#F5F0E8` | `Nomad.warmCard` | Trip preview cards inside ProfileSheet, stats row background |
| Accent (10%) | `#E8A44A` | `Nomad.amber` | See reserved list below |
| Destructive | `#D94F3D` | `Nomad.destructive` | Not used in Phase 3 (no destructive actions) |

Accent (`Nomad.amber`) reserved for in Phase 3:
1. Trip card route preview Path stroke line (amber line on dark/cream strip)
2. Recording pill pulsing dot (solid amber-tinted red â€” see note below)
3. "Stop Trip" button background on recording pill
4. Numbered place pin marker fill (amber circle with white number)
5. Globe trip pinpoints (inherited from Phase 1 â€” unchanged)
6. Drag strip pill handle indicator (amber-tinted, subtle)
7. Key stat values in TripDetailSheet (e.g., step count, distance â€” amber foreground color)
8. Active trip indicator's "+" CTA in ProfileSheet

**Note on recording dot:** The pulsing indicator uses `Color.red` (system red) for the dot itself â€” not `Nomad.amber` â€” to signal urgency/active state. The amber is used for the Stop Trip button. Do NOT use `Nomad.destructive` for this dot; use `Color.red` (system).

Do NOT use amber for: body text, captions, date strings, stat labels, non-interactive decorative elements, map polyline (use amber for polyline â€” see below).

**Route polyline color:** `Nomad.amber` stroke, 3pt line width (CONTEXT.md D-13). This is the only amber use on the MapKit map surface.

## Interaction Contract

| Interaction | Behavior |
|-------------|---------|
| Tap drag strip | ProfileSheet slides up. Spring: damping 0.85, response 0.4. |
| Drag strip upward | ProfileSheet opens as drag progresses. Native `.sheet` detent behavior. |
| Tap "+" in ProfileSheet | `LocationManager.startRecording(tripId:)` called immediately. ProfileSheet dismisses. Recording pill appears on globe with 0.3s fade-in. |
| Tap "Stop Trip" on pill | Trip name `UIAlertController` presented. Pill remains visible. |
| Save Trip in dialog | `TripService.finalizeTrip(...)` called. Pill fades out (0.3s). ProfileSheet reopens showing new trip at top. |
| Discard Trip in dialog | `LocationManager.stopRecording()` called. SwiftData route points for this tripId purged. Pill fades out (0.3s). |
| Tap trip card | TripDetailSheet slides up over ProfileSheet. Spring: damping 0.85, response 0.4. |
| Swipe down TripDetailSheet | TripDetailSheet dismisses. ProfileSheet remains visible and interactive. |
| Swipe down ProfileSheet | ProfileSheet dismisses. Globe is full-screen and interactive. |
| Tap globe pinpoint | ProfileSheet opens scrolled to matching trip. |
| 3rd dismissed auto-detect notification | `VisitMonitor.stopMonitoring()` called. `manualOnlyMode = true` in UserDefaults. No visible UI change. |
| Auto-detect notification tap | Opens app; if app is foregrounded, no action needed (user can tap "+" manually). |
| Photo thumbnail tap | No action (disabled, CONTEXT.md D-18). |
