## Color

All colors use the established `Color.Nomad` namespace. No new color values are introduced in Phase 3.1.

| Role | Hex | SwiftUI Name | Usage |
|------|-----|-------------|-------|
| Globe Background | `#0A0A14` | `Nomad.globeBackground` | Sheet background (dark aesthetic); text foreground on light surfaces |
| Dominant (60%) | `#FAF8F4` | `Nomad.cream` | Sheet surface background (via `.panelGradient()`); temperature pill foreground text |
| Secondary (30%) | `#F5F0E8` | `Nomad.warmCard` | City thumbnail card backgrounds; stats pill background; trip log card backgrounds |
| Accent (10%) | `#E8A44A` | `Nomad.amber` | See reserved list below |
| Destructive | `#D94F3D` | `Nomad.destructive` | Not used in Phase 3.1 (no destructive actions) |

Accent (`Nomad.amber`) reserved for in Phase 3.1:
1. Selected city thumbnail card border/ring indicator (2pt border when active)
2. Stats pill stat values (numeric foreground)
3. Back/dismiss chevron icon tint
4. City strip active indicator dot below selected city card (4pt amber dot)
5. Temperature notch pill background (use `Nomad.globeBackground` — see note below)

**Note on temperature notch color:** The temperature pill uses `Nomad.globeBackground` (dark `#0A0A14`) at 90% opacity as background — NOT `Nomad.amber`. Text inside is `Nomad.cream`. This matches the dark pill in the reference screenshot. Amber is NOT used as the pill background.

**Photo card background:** Photos are user PHAssets rendered full-width. No background color is needed when a photo loads. Use `Nomad.warmCard` as the placeholder while the image loads.

**Sheet aesthetic note:** Phase 3.1 uses the same `.panelGradient()` modifier as all other sheets. The dark globe aesthetic (CONTEXT.md "dark background, consistent with existing Nomad design") is achieved via the panel gradient's corner grain effect on `Nomad.cream`, not by switching to a dark sheet background. The sheet background stays `Nomad.cream` (60% dominant).

Source: AppColors.swift (Phase 1); CONTEXT.md decisions, specifics ("dark pill with temperature value"); Phase 3 UI-SPEC color section; design description ("dark background, consistent with existing Nomad design")

## New Visual Components (Phase 3.1)

### Country Detail Sheet — Container

Slides up from the bottom panel when a visited country is tapped on the globe.

| Property | Specification |
|----------|--------------|
| Presentation | SwiftUI `.sheet` nested appropriately in GlobeView, or presented over GlobeView. Detent: `.large` (full-height). `presentationDragIndicator(.visible)` |
| Dismiss | Swipe down (system drag-to-dismiss). Back chevron top-left also dismisses. Globe returns to full-screen interactive state on dismiss. |
| Background | `.panelGradient()` — same as ProfileSheet and TripDetailSheet |
| Scroll | `ScrollView` (.vertical) wrapping `VStack`. City strip stays sticky at top until user scrolls past it, then scrolls with content. Note: sticky behavior may require `coordinateSpace` tracking; simpler alternative is to not make city strip sticky — place it in the scroll flow naturally. Use non-sticky layout. |
| Panel drag indicator | System `.presentationDragIndicator(.visible)` — 4pt × 36pt capsule, system color |

Source: CONTEXT.md decisions (entry point, presentation, dismiss); Phase 3 UI-SPEC (TripDetailSheet pattern)

### City Strip (Horizontal Scroll)

Immediately below the country header. Shows one thumbnail card per visited city, ordered by geographic proximity clustering.

| Property | Specification |
|----------|--------------|
| Layout | Horizontal `ScrollView` (.horizontal, showsIndicators: false) with `HStack(spacing: 8)` |
| Horizontal padding | 16pt leading + trailing (applied via `.padding(.horizontal, 16)` on HStack) |
| Height | 100pt fixed (80pt thumbnail + 4pt gap + 13pt caption label line height + 3pt padding) |
| Scroll behavior | `.scrollTargetBehavior(.viewAligned)` is NOT needed — free scroll |

**City Thumbnail Card:**

| Property | Specification |
|----------|--------------|
| Card size | 64pt wide × 80pt tall (image area); full card with label = 64 × 100pt |
| Image | `PHAsset` first-matched photo for that city, `.fill` content mode, clipped to 64×80pt RoundedRectangle (10pt corner radius) |
| Image placeholder | `Nomad.warmCard` while loading |
| City label | `AppFont.caption()` — 13pt Inter Regular, `Nomad.globeBackground` foreground, centered below image, 4pt top gap |
| Label truncation | Single line, `.lineLimit(1)`, `.truncationMode(.tail)` |
| Selected state border | 2pt amber (`Nomad.amber`) RoundedRectangle stroke overlay on the image area |
| Selected state dot | 4pt diameter filled circle, `Nomad.amber`, centered below the city label, 4pt below label |
| Unselected state | No border, no dot |
| Tap behavior | Sets `selectedCityIndex`, scrolls photo carousel to corresponding page with `.animated` behavior |
| Tap target | Full 64×100pt card area |

Source: CONTEXT.md decisions (city strip, horizontal scroll, tapping a city card scrolls carousel); design description (cities grouped by geographic proximity)

### Photo Carousel (Full-Width, Per City)

Full-width paged carousel below the city strip. One page per city. Shows user's PHAsset photos for that city.

| Property | Specification |
|----------|--------------|
| Layout | `TabView(selection: $selectedCityIndex)` with `.tabViewStyle(.page(indexDisplayMode: .never))`. One page per city. |
| Page width | Full screen width (no horizontal padding — edge to edge) |
| Photo card height | 320pt fixed (tall enough for portrait photos to feel cinematic) |
| Photo content mode | `.fill`, clipped to 320pt height. Corner radius: 0pt on left/right edges (edge-to-edge); 0pt (truly full bleed). |
| Photo placeholder | `Nomad.warmCard` at 320pt height while loading |
| Multiple photos per city | Show first matched photo from that city's PHAssets. If multiple, do NOT paginate within a city page (single representative photo per city card is sufficient for Phase 3.1). |
| Page transition | System `TabView` page swipe. Matches swipe to city card selection. |
| Tap behavior | Disabled (no tap action in Phase 3.1) |

**Temperature Notch:**

| Property | Specification |
|----------|--------------|
| Position | Centered horizontally, overlaid at the top of the photo card. `offset(y: -14)` from photo card top edge places pill half-above, half-below the top edge. |
| Pill shape | `Capsule()` |
| Pill width | Hug content + 12pt horizontal padding each side |
| Pill height | 28pt |
| Background | `Nomad.globeBackground` at 90% opacity |
| Border | 1pt stroke, `Nomad.cream` at 20% opacity |
| Text | Temperature value + unit, e.g. "−8°C". `AppFont.caption()` — 13pt Inter Regular, `Nomad.cream` foreground |
| Temperature source | WeatherKit current conditions (primary); fallback to historical temperature stored on `TripDocument` if WeatherKit unavailable (Claude's Discretion: implementation strategy) |
| Loading state | Show "—°C" until temperature resolves |
| Unavailable state | Omit pill entirely (do not show "N/A") |

Source: CONTEXT.md decisions (photo carousel, temperature notch, dark pill); design description ("dark pill centered at top of photo card showing temperature")

### Stats Pill

Single horizontal pill row below the location identity block.

| Property | Specification |
|----------|--------------|
| Position | 16pt top gap below location identity block |
| Horizontal padding | 16pt (pill fills between left/right padding) |
| Shape | `RoundedRectangle(cornerRadius: 22)` (pill) |
| Background | `Nomad.warmCard` |
| Height | 44pt (vertical padding: 8pt top + bottom + content) |
| Layout | `HStack(spacing: 0)` with three `statCell` views and two `Divider` separators |
| Divider | `Rectangle().fill(Nomad.globeBackground.opacity(0.15))`, 1pt wide, 20pt tall, centered vertically |
| Stat cell | `VStack(spacing: 2)`: value (top) + label (bottom), `frame(maxWidth: .infinity)` |
| Value style | `AppFont.body()` — 16pt Inter Regular, `Nomad.amber` foreground |
| Label style | `AppFont.caption()` — 13pt Inter Regular, `Nomad.globeBackground` at 60% opacity |

Three stats displayed (left to right):

| Stat | Value Format | Label |
|------|-------------|-------|
| Trip logs count | "{N}" (integer, no grouping separator needed) | "logs" |
| Distance traveled | "{N.N} km" (one decimal place) | "km" |
| Photos taken | "{N}" (integer) | "photos" |

Source: CONTEXT.md decisions (stats pill: logs · km · photos); design description (adapt "26 ski runs · 112 km · 8 types" to Nomad data model)

## Interaction Contract

| Interaction | Behavior |
|-------------|---------|
| Tap visited country on globe | Country Detail Sheet slides up. `.sheet` presentation, `.large` detent. Spring animation: system default sheet spring. |
| Tap back chevron | Sheet dismisses. Globe returns to full-screen interactive state. |
| Swipe sheet down | Sheet dismisses (system drag-to-dismiss). Same result as back chevron. |
| Tap city thumbnail card | `selectedCityIndex` updates. Photo carousel scrolls to that city page (`.animated`). Selected card shows 2pt amber border + amber dot indicator. |
| Swipe photo carousel left/right | `selectedCityIndex` updates. City strip does NOT auto-scroll to keep selected card visible — static strip position. |
| Tap trip log card | `TripDetailSheet` slides up over Country Detail Sheet (nested per INFRA-02). Spring: damping 0.85, response 0.4. |
| Swipe down TripDetailSheet | TripDetailSheet dismisses. Country Detail Sheet remains visible and interactive. |
| Photo area tap | Disabled — no gesture recognizer on photo card or carousel. |
| Temperature notch | Display only — no tap action. |
| Stats pill | Display only — no tap action. |

Source: CONTEXT.md decisions (entry point, dismiss, city strip tap, carousel scroll); INFRA-02 (nested sheet, no cascade); Phase 3 UI-SPEC interaction contract (TripDetailSheet nested pattern)

## State Summary

| State | Trigger | Visual |
|-------|---------|--------|
| Sheet loading | On appear, before Firestore fetch completes | `ProgressView()` centered below country header, city strip skeleton (3 × `Nomad.warmCard` rounded rects 64×80pt) |
| Sheet loaded | Firestore city data resolved | Full city strip + first city's photo carousel visible |
| No cities | Country in `visitedCountryCodes` but 0 trips | Empty state copy below header |
| City selected | User taps city card or swipes carousel | Amber border on city card; carousel page shows that city |
| Photo loading | PHAsset fetch in progress for selected city | `Nomad.warmCard` at 320pt height (placeholder) |
| Temperature loading | WeatherKit fetch in progress | "—°C" in pill |
| Temperature unavailable | WeatherKit error or no data | Pill hidden |
| No photos for city | 0 PHAssets matched | Placeholder card with "No photos for {CityName}." centered |
| No trip logs for city | 0 TripDocuments for city | "No trips logged for {CityName}." in trip logs section |

Source: CONTEXT.md decisions (Claude's Discretion: empty states); Phase 3 UI-SPEC (loading/error patterns)
