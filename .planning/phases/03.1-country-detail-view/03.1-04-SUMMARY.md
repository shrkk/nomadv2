# Phase 03.1 Plan 04: Globe Integration + End-to-End Verification Summary

**One-liner:** CountryDetailSheet wired into GlobeView via showCountryDetailSheet routing in GlobeViewModel — country tap now presents CountryDetailSheet; JourneyPill visibility controlled by sheet state.

## What Was Built

### Task 1: Add country detail state to GlobeViewModel and wire GlobeView

**`Nomad/Globe/GlobeViewModel.swift`** — three additions:
- `var showCountryDetail = false` — drives CountryDetailSheet presentation
- `var selectedCountryCode: String? = nil` — stores the tapped country code
- `var trips: [TripDocument] = []` — stub-ready trip list for CountryDetailSheet (empty until Firebase loading is wired)
- `func showCountryDetailSheet(code:)` — sets `showProfileSheet = false` first (T-3.1-09 mutual exclusion), then sets `selectedCountryCode`, `focusedCountryCode`, `showPinpoints = true`, `showCountryDetail = true`
- `animateToCountry(code:)` now delegates entirely to `showCountryDetailSheet(code:)`

**`Nomad/Globe/GlobeView.swift`** — four changes:
- Added CountryDetailSheet `.sheet(isPresented: $viewModel.showCountryDetail)` modifier after ProfileSheet modifier — filters `viewModel.trips` by `visitedCountryCodes.contains(code)` and passes to `CountryDetailSheet(countryCode:trips:)`
- JourneyPill added to ZStack body — hidden when `showProfileSheet || showCountryDetail` (per RESEARCH.md recommendation)
- `handleTap` in Coordinator now additionally guards `visitedCodes.contains(code)` before calling `onTapCountry` (T-3.1-08 mitigation)
- Removed orphaned trip-recording methods (`presentTripNameAlert`, `saveTrip`, `discardTrip`, `queryStepCount`, `calculateDistance`) that had no property declarations — these were a stash conflict artifact from commit 604e0a7
