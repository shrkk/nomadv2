# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: Data & Auth Foundation.
> Covers the complete onboarding flow: Welcome → Sign Up → Handle → Location permission → Photos permission → Discovery scope → Home city confirm → Globe.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (native iOS — SwiftUI + UIKit where needed) |
| Preset | not applicable |
| Component library | SwiftUI native components |
| Icon library | SF Symbols 5 (system icons only) |
| Font | Playfair Display (display/heading) + Inter (body/labels/buttons) |

**Note:** This is a native iOS Swift application. shadcn/Tailwind/Radix are not applicable. All design tokens inherit directly from the `AppFont`, `AppColors`, and `PanelGradient` design system established in Phase 1. No new tokens are introduced — Phase 2 extends the existing system.

Source: Phase 1 UI-SPEC (approved), `AppFont.swift`, `AppColors.swift`, `PanelGradient.swift`

## Spacing Scale

Inherited from Phase 1 — unchanged. All values in SwiftUI points (1pt = 1px on non-retina; scales on retina).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4pt | Icon gaps, inline label padding, validation icon spacing |
| sm | 8pt | Compact element spacing, text field label-to-field gap |
| md | 16pt | Default element spacing, onboarding screen horizontal padding |
| lg | 24pt | Section padding, between form fields |
| xl | 32pt | Layout gaps, above/below primary CTA |
| 2xl | 48pt | Bottom safe-area inset for CTA buttons |
| 3xl | 64pt | Top safe-area clearance on Welcome screen |

Exceptions:
- Touch targets (buttons, option cards, text fields): minimum 44pt hit area — per Apple HIG
- Discovery scope option cards: minimum 56pt tall to accommodate icon + label + description line
- Text field height: 48pt (taller than default to reduce mis-tap rate on onboarding forms)
- Progress indicator dots: 8pt diameter dots with 4pt gap between each dot (xs token gap)

Source: Inherited from Phase 1 UI-SPEC (spacing scale); exceptions derived from CONTEXT.md D-06 (option cards) and Apple HIG touch target guidelines.

## Typography

Identical to Phase 1 contract. All type via `AppFont` enum. No new sizes or weights introduced in Phase 2.

| Role | SwiftUI Size | Font Face | Weight | Line Height | Usage in Phase 2 |
|------|-------------|-----------|--------|-------------|-----------------|
| Title | 28pt | Playfair Display | Semibold (600) | 1.2 | Screen headlines (e.g., "Welcome to Nomad", "Create your account", "Choose your handle") |
| Subheading | 20pt | Playfair Display | Regular (400) | 1.3 | Tagline on Welcome screen, section labels in permission pre-prompts |
| Body | 16pt | Inter | Regular (400) | 1.5 | Explanatory copy on all onboarding screens, form field placeholder text, option card descriptions |
| Caption | 13pt | Inter | Regular (400) | 1.4 | Inline validation messages (handle available/taken, password requirements), progress step labels |

Button labels: 16pt Inter Semibold (600) — same size as Body, Semibold weight. Not a fifth size; reuses Body size slot. All CTAs ("Get started", "Continue", "Confirm", "Create account") use this token.

Source: Phase 1 UI-SPEC Typography section; `AppFont.swift` canonical implementation.

## Color

All colors via `Color.Nomad` namespace established in Phase 1. No new colors introduced.

| Role | Hex | SwiftUI Name | Usage |
|------|-----|-------------|-------|
| Dominant (60%) | `#FAF8F4` | `Nomad.cream` | All onboarding screen backgrounds, text field surfaces |
| Secondary (30%) | `#F5F0E8` | `Nomad.warmCard` | Discovery scope option card backgrounds (unselected), home city confirmation card |
| Accent (10%) | `#E8A44A` | `Nomad.amber` | See reserved list below |
| Destructive | `#D94F3D` | `Nomad.destructive` | Sign-in error inline messages only (invalid credentials, account not found) |
| Globe Background | `#0A0A14` | `Nomad.globeBackground` | Welcome screen background behind the globe hero visual |

Accent (`Nomad.amber`) reserved for in Phase 2:
1. "Get started" CTA button background on Welcome screen
2. "Continue" / "Confirm" / "Create account" primary button backgrounds on all onboarding steps
3. Selected state border/fill on Discovery scope option cards (selected card: amber border 2pt, amber background at 10% opacity)
4. Handle availability checkmark SF Symbol tint (checkmark.circle.fill) when handle is available
5. Progress indicator dots — filled/active dot uses amber; inactive dots use `Nomad.warmCard`
6. Home city confirmation "Confirm" button

Do NOT use amber for: body text, captions, validation error messages, unselected option card borders, text field borders (unless focused — see Interaction Contract).

Text field focused-state border: `Nomad.amber` at 60% opacity, 1pt border. Unfocused: `Nomad.warmCard` at 100%, 1pt border. Error state: `Nomad.destructive` at 80%, 1pt border.

Source: `AppColors.swift` (canonical); CONTEXT.md D-06 (option card selection state); Phase 1 UI-SPEC Color section.

## Onboarding Flow — Screen-by-Screen Contract

8 screens in a single full-screen paged flow. Each screen occupies 100% of the display. Progress is indicated by dots at the top of screens 2–7 (not on Welcome; not on the Globe exit screen).

Source: CONTEXT.md D-01, D-02, D-03 (flow order and presentation decisions).

### Screen 1: Welcome

**Purpose:** First impression — globe as hero, single CTA.

| Property | Specification |
|----------|--------------|
| Background | `Nomad.globeBackground` (`#0A0A14`) — full bleed, matches the globe environment |
| Globe visual | The live RealityKit globe (from Phase 1) rendered behind the content layer. Not a screenshot. |
| Tagline | "The world is yours to explore." — `AppFont.subheading()` (20pt Playfair Display Regular), `Color.white` at 85% opacity, center-aligned, positioned at 40% screen height |
| App name | "Nomad" — `AppFont.title()` (28pt Playfair Display Semibold), `Color.white`, center-aligned, 16pt above tagline |
| CTA button | "Get started" — full-width (md padding each side = 16pt margins), 48pt tall, `Nomad.amber` background, `Nomad.globeBackground` text via `AppFont.buttonLabel()`, 12pt corner radius |
| CTA position | Pinned to bottom: 48pt above the bottom safe area edge (2xl token) |
| Sign in link | "Already have an account? Sign in" — `AppFont.caption()` (13pt Inter Regular), `Color.white` at 60% opacity, centered, 16pt below CTA button. "Sign in" portion is tappable — tap routes to Sign Up screen with email/password pre-filled in sign-in mode. |
| Transition in | No entry animation — this is the initial screen |
| Transition out | Horizontal slide left (next screen slides in from right). Spring: damping 0.85, response 0.4. |

### Screen 2: Sign Up

**Purpose:** Email and password entry.

| Property | Specification |
|----------|--------------|
| Background | `Nomad.cream` (`#FAF8F4`) — full bleed |
| Header | "Create your account" — `AppFont.title()`, `Nomad.globeBackground`, top-aligned with 32pt top padding after safe area |
| Subheader | "We'll keep your journey private and secure." — `AppFont.body()`, `Nomad.globeBackground` at 60% opacity, 8pt below header |
| Email field | Standard iOS TextField, 48pt tall, `Nomad.warmCard` background, 12pt corner radius, "Email address" placeholder in `AppFont.body()` at `Nomad.globeBackground` 40% opacity. Keyboard: `.emailAddress`. Autocapitalization: `.none`. |
| Password field | Standard iOS SecureField, same dimensions as email. "Password" placeholder. Trailing show/hide toggle: `eye` / `eye.slash` SF Symbol at 16pt, `Nomad.globeBackground` 40% opacity. |
| Password hint | "8+ characters" — `AppFont.caption()`, `Nomad.globeBackground` at 50% opacity, 4pt below password field |
| Error state | Inline below the offending field: red text via `AppFont.caption()`, `Nomad.destructive`, 4pt gap. Field border switches to `Nomad.destructive` at 80% opacity. |
| CTA | "Create account" — same button spec as Welcome CTA. Disabled (50% opacity) until both fields are non-empty. |
| CTA position | 32pt below password hint; not pinned to bottom — scrollable if keyboard pushes content |
| Progress dots | 6 dots at top (16pt from safe area), dot 1 active (amber), dots 2–6 inactive (warmCard). |

### Screen 3: Handle

**Purpose:** Set unique @handle with live validation.

| Property | Specification |
|----------|--------------|
| Background | `Nomad.cream` — full bleed |
| Header | "Choose your handle" — `AppFont.title()`, `Nomad.globeBackground` |
| Subheader | "This is how other travelers will find you." — `AppFont.body()`, `Nomad.globeBackground` at 60% opacity |
| Handle field | TextField with "@" prefix label in `AppFont.body()` at `Nomad.globeBackground` 60% opacity. Field text: `AppFont.body()`, `Nomad.globeBackground`. Autocapitalization: `.none`. Autocorrection: off. Keyboard: `.asciiCapable`. Character set: lowercase letters, digits, underscores only (enforce via input filtering). Max length: 30 characters. |
| Validation states | 3 states with a trailing indicator inside the field: |
| — Idle | No indicator |
| — Checking | `ProgressView` (small spinner), `Nomad.globeBackground` at 50% opacity. Fires 500ms after the user stops typing. |
| — Available | `checkmark.circle.fill` SF Symbol, `Nomad.amber`. "Available" text via `AppFont.caption()`, `Nomad.amber`, 4pt below field. |
| — Taken | `xmark.circle.fill` SF Symbol, `Nomad.destructive`. "Already taken" text via `AppFont.caption()`, `Nomad.destructive`, 4pt below field. |
| — Invalid format | `exclamationmark.circle.fill` SF Symbol, `Nomad.destructive`. "Letters, numbers, and underscores only" via `AppFont.caption()`, `Nomad.destructive`. |
| CTA | "Continue" — same button spec. Enabled only when state is "Available". |
| Progress dots | Dot 2 active. |

### Screen 6: Discovery Scope

**Purpose:** Choose when the app should detect trips.

| Property | Specification |
|----------|--------------|
| Background | `Nomad.cream` — full bleed |
| Header | "When should Nomad log trips?" — `AppFont.title()`, `Nomad.globeBackground` |
| Subheader | "You can change this anytime in settings." — `AppFont.caption()`, `Nomad.globeBackground` at 50% opacity |
| Option cards | Two cards, full-width (md = 16pt horizontal margins), stacked with 16pt gap |
| Card height | 88pt minimum — accommodates icon (24pt) + label (16pt) + description (13pt) with internal padding |
| Card surface | Unselected: `Nomad.warmCard` background, 1pt border `Nomad.warmCard` (border matches fill — appears borderless), 12pt corner radius |
| Card selected | `Nomad.amber` background at 10% opacity, 2pt border `Nomad.amber`, 12pt corner radius |

| Card 2 — "Away from home only" | Icon: `house` SF Symbol. Label: "Away from home only" — same label spec. Description: "Only log trips when you leave your home city." — `AppFont.caption()`, same. This card is the pre-selected default. |

### Screen 7: Home City Confirm

**Purpose:** Detect and confirm home city for geofence registration.

| Property | Specification |
|----------|--------------|
| Background | `Nomad.cream` — full bleed |
| Header | "Your home base" — `AppFont.title()`, `Nomad.globeBackground` |
| Body | "We've detected your home city. Confirm it's right so we know when you've set off on an adventure." — `AppFont.body()`, `Nomad.globeBackground` at 75% opacity |
| Detection state | While CLGeocoder is running: show a `ProgressView` (small spinner, `Nomad.amber`) centered below the body text. CTA is hidden during detection. |
| City display | After detection: a confirmation card — `Nomad.warmCard` background, 12pt corner radius, 16pt internal padding. Card shows: `mappin.circle.fill` SF Symbol (24pt, `Nomad.amber`) + detected city name via `AppFont.subheading()` (20pt Playfair Display Regular) + "Your home city" label via `AppFont.caption()`, `Nomad.globeBackground` at 50% opacity. |
| Edit affordance | "That's not right" text button below the card — `AppFont.caption()`, `Nomad.amber`, underline style. Tapping it reveals an inline TextField replacing the city display card. TextField: same spec as Handle field but keyboard type `.default`, autocapitalization `.words`. User types a city name; no live search — they type and continue. |
| CTA | "Confirm home city" — same button spec. Enabled after detection completes (auto-enabled with detected city; also enabled if user has typed in the edit field). |
| Progress dots | Dot 6 active. |
| Failure state | If CLGeocoder fails (no location permission granted, or network error): show "Could not detect your city" via `AppFont.body()`, `Nomad.globeBackground` at 60% opacity. Edit text field appears immediately. CTA reads "Set home city manually". |

### Screen 8: Globe (Post-Onboarding Exit)

No onboarding UI — the flow transitions directly to `GlobeView` after the home city is confirmed and onboarding is written to Firestore (`onboardingComplete: true`). No splash screen, no "you're all set" interstitial. The globe is the reward.

Source: CONTEXT.md D-11 (no splash screen for returning users also applies to onboarding exit).

## Onboarding Navigation Contract

| Property | Specification |
|----------|--------------|
| Transition between screens | Horizontal slide: current screen slides left, next screen enters from right. Spring: damping 0.85, response 0.4. Consistent with sheet animations established in Phase 1. |
| Back navigation | Not available on Welcome (first screen). On screens 2–7: a back chevron (`chevron.left`) at top-left, 44pt hit area, `Nomad.globeBackground` at 70% opacity. Back navigates to prior screen with reverse slide (current exits right, prior enters from left). |
| Progress dots position | Centered horizontally, 16pt below the top safe area edge. 8pt diameter per dot, 4pt gap between dots. Active dot: `Nomad.amber` filled circle. Inactive dot: `Nomad.warmCard` filled circle. Total dot count = 6 (screens 2–7). |
| Keyboard avoidance | All screens with text fields use `.ignoresSafeArea(.keyboard, edges: .bottom)` with the view content scrollable — fields must be visible when keyboard is up. |
| Permission denial recovery | Onboarding never blocks on permission denial (location, photos). The user always reaches the globe. Denied permissions are surfaced in Phase 3 as contextual nudges, not at onboarding time. |

Source: CONTEXT.md D-01 (full-screen paged flow, one-thing-at-a-time), D-05 (permission screens), D-04 (live handle validation debounce).

## Auth State & Loading Contract

| State | Visual |
|-------|--------|
| `.loading` | Stay on launch screen (iOS LaunchScreen.storyboard). No custom overlay. Firebase resolves in milliseconds for cached sessions — this state is rarely visible. |
| `.unauthenticated` | `OnboardingView` — Welcome screen (Screen 1). |
| `.authenticated(user)` + `onboardingComplete: true` | `GlobeView` — no transition animation; globe appears directly. |
| `.authenticated(user)` + `onboardingComplete: false` | `OnboardingView` — resume from last incomplete step (use `onboardingStep` field on user document or local UserDefaults flag). |

Source: CONTEXT.md D-09, D-10, D-11.

## Interaction Contract

| Interaction | Behavior |
|-------------|---------|
| Tap "Get started" | Slide to Sign Up (Screen 2). Spring: damping 0.85, response 0.4. |
| Tap "Sign in" link on Welcome | Slide to Sign Up screen in sign-in mode (email + password fields, CTA reads "Sign in", no "Create account" framing). |
| Tap back chevron | Reverse slide to prior screen. |
| Type in Handle field | 500ms debounce fires Firestore uniqueness check. Spinner shows during async check. Result updates indicator inline — no sheet, no alert. |
| Tap "Enable background location" | Fires `CLLocationManager.requestAlwaysAuthorization()`. Native iOS dialog appears. After dialog closes (any result): advance to Screen 5. |
| Tap "Connect your photos" | Fires `PHPhotoLibrary.requestAuthorization(for: .readWrite)`. Native iOS dialog appears. After dialog closes (any result): advance to Screen 6. |
| Tap Discovery scope card | Deselects current selection, selects tapped card. Tap animation: scale to 0.97 with spring, release back to 1.0. |
| Tap "That's not right" on Home City | City card fades out (100ms), edit TextField fades in (100ms). Keyboard appears. |
| Tap "Confirm home city" | Registers CLCircularRegion geofence (50km radius). Writes user document to Firestore. Sets `onboardingComplete: true`. Transitions to GlobeView with a cross-dissolve (0.3s). |
| Network/Firestore failure on "Confirm home city" | Retry is silent — no error sheet. Show an inline error below the CTA: "Couldn't save. Tap to retry." The CTA tap retries the Firestore write. |
| Returning authenticated user (onboardingComplete: true) | ContentView routes directly to GlobeView. No Welcome screen shown. |
