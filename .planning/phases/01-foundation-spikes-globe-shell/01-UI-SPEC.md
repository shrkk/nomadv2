## Spacing Scale

Declared values (multiples of 4, in SwiftUI points — 1pt = 1px on non-retina, scales on retina):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4pt | Icon gaps, inline label padding |
| sm | 8pt | Compact element spacing within cards |
| md | 16pt | Default element spacing, panel internal padding |
| lg | 24pt | Section padding, between card rows |
| xl | 32pt | Layout gaps, panel header spacing |
| 2xl | 48pt | Major section breaks |
| 3xl | 64pt | Page-level / full-bleed spacing |

Exceptions:
- Touch targets (buttons, pinpoints, interactive SF Symbols): minimum 44pt hit area regardless of visual size — per Apple HIG
- Globe pinpoints: 44pt hit area with a 20pt visual dot
- Bottom sheet drag handle: 44pt tall interaction strip even if handle mark is 4pt tall

Source: spacing scale is standard iOS 8-point grid; touch target exceptions per Apple HIG and CONTEXT.md D-03 (pinpoints on globe surface)

## Typography

All type is rendered via a shared `AppFont` enum with static properties. Playfair Display is loaded as a bundled custom font. Inter is loaded as a bundled custom font (Inter Variable preferred; fallback: system Inter via `.design(.rounded)` is NOT acceptable — must use bundled Inter).

Two weights are declared for the entire scale:
- **Regular (400)** — body text, captions, subheadings
- **Semibold (600)** — titles, button labels, emphasized stats

Both Playfair Display and Inter are loaded at weight 600 (Semibold) and weight 400 (Regular). Playfair Display Semibold and Inter Semibold are both resolved as 600 — they are the same semantic weight token, not separate weights.

| Role | SwiftUI Size | Font Face | Weight | Line Height |
|------|-------------|-----------|--------|-------------|
| Title | 28pt | Playfair Display | Semibold (600) | 1.2 |
| Subheading | 20pt | Playfair Display | Regular (400) | 1.3 |
| Body | 16pt | Inter | Regular (400) | 1.5 |
| Caption | 13pt | Inter | Regular (400) | 1.4 |

Button labels use 16pt Inter Semibold (600) — same size as Body, different weight. This is not a fifth size; it reuses the Body size slot with the Semibold weight.

Source: D-08 (CONTEXT.md) for sizes — LargeTitle (34pt) removed to comply with 4-size maximum; Title (28pt) becomes the top of the scale. Weights are Claude's Discretion (CONTEXT.md §Claude's Discretion) — consolidated from 3 values to 2 to comply with 2-weight maximum.

## Color

The color scheme is a warm editorial palette: cream dominates, amber accents, space-black for the globe background. All colors are defined as SwiftUI `Color` extensions on `Color.Nomad` namespace.

| Role | Hex | SwiftUI Name | Usage |
|------|-----|-------------|-------|
| Globe Background | `#0A0A14` | `Nomad.globeBackground` | RealityKit scene background, deep space black with blue tint |
| Dominant (60%) | `#FAF8F4` | `Nomad.cream` | Panel backgrounds, sheet surfaces, primary view backgrounds |
| Secondary (30%) | `#F5F0E8` | `Nomad.warmCard` | Card surfaces inside panels, inner containers |
| Accent (10%) | `#E8A44A` | `Nomad.amber` | See reserved list below |
| Destructive | `#D94F3D` | `Nomad.destructive` | Destructive actions only (not applicable in Phase 1) |
| Star field | `#FFFFFF` at 15–40% opacity | `Nomad.star` | Globe background star particles |
| Country glow | `#E8A44A` at 60% opacity + 2–3px outer glow | derived from `Nomad.amber` | Visited country polygon overlays |

Accent (`Nomad.amber`) reserved for:
1. Visited country polygon fills on globe surface (60% opacity)
2. Country polygon outer glow (2–3pt bleed, ~30% opacity)
3. Panel gradient bleed corners (amber fading to `Nomad.cream`, not solid fill)
4. Interactive button labels and primary call-to-action backgrounds
5. Key stat values in the globe home view (e.g., hardcoded country count stub)
6. SF Symbol tint for interactive globe pinpoints

Do NOT use amber for: body text, captions, non-interactive decorative elements, panel body backgrounds.

Globe background `#0A0A14` is used only for the RealityKit/ARView scene. Panel and SwiftUI layers always use `#FAF8F4`.

Source: D-05, D-06 (CONTEXT.md); globe background derived from D-01 ("deep black/navy")

## Panel Gradient

Every bottom sheet panel in this phase uses the same gradient treatment. This becomes the canonical pattern for all panels in Phases 2–4.

- **Gradient:** Warm amber (`#E8A44A` at 20% opacity) radiates from top-left corner and top-right corner of the panel, fading to transparent by the midpoint of the panel. The fade direction is radial from corner, not linear top-to-bottom.
- **Base surface:** `#FAF8F4` (Nomad.cream) — solid, no transparency.
- **Grain texture:** Monochrome noise overlay at 8% opacity applied over the gradient. Implementation method is Claude's Discretion (Metal shader preferred for performance; CoreImage or a bundled PNG fallback is acceptable).
- **Corner radius:** 20pt on top-left and top-right corners of the sheet. Bottom corners: 0pt (sheet extends to safe area edge).
- **Panel handle:** 4pt tall × 36pt wide pill, `Nomad.warmCard` (`#F5F0E8`), centered horizontally, 8pt from top edge. 44pt tall interaction strip.

Source: D-07 (CONTEXT.md); grain percentage and corner radius are Claude's Discretion defaults.

## Globe Environment

| Property | Specification |
|----------|--------------|
| Background | Deep space: `#0A0A14`, fills full RealityKit viewport |
| Star field | White particles, 15–40% opacity, density and size at Claude's Discretion |
| Lighting | One primary directional light, warm hemisphere (warm tint on visible side, cool dark on reverse). No ambient fill light — keep the dark side dark. |
| Globe rotation | Free pan and pinch-zoom under touch. No inertia decay limit — natural SwiftUI gesture feel. |
| Country fills | Visited countries: `#E8A44A` at 60% opacity, projected as GeoJSON polygons onto sphere surface |
| Country glow | 2–3pt outer glow from polygon edge, `#E8A44A` at 30% opacity |
| Country animation | Static — no pulse, no transition animation on highlight. Render state is set once at load. |
| Pinpoints | Amber `#E8A44A` dots, 20pt visual size, 44pt hit area, positioned at trip centroid coordinates |

Source: D-01, D-02, D-03, D-04 (CONTEXT.md)

## Interaction Contract

| Interaction | Behavior |
|-------------|---------|
| Globe pan | One-finger drag rotates globe on Y axis (yaw) and X axis (pitch). No snap or limit. |
| Globe pinch | Two-finger pinch zooms camera distance. Min zoom: full-globe view. Max zoom: continent-level (~30% of full globe radius visible). |
| Tap highlighted country | Globe camera animates smoothly to center that country's centroid, zooms to region level. Trip pinpoints appear. Duration: 600ms ease-in-out. |
| Tap pinpoint | Profile bottom sheet slides up. Sheet origin: bottom of screen. Animation: spring with damping 0.85, response 0.4. |
| Tap outside sheet | Sheet dismisses by sliding down. Globe returns to free-rotate state. |
| Swipe sheet down | Sheet dismisses. Same animation reversed. |
| Tap trip card in profile sheet | Second (trip detail) sheet slides up over profile sheet. Profile sheet remains visible behind. |
| Dismiss trip detail sheet | Slide down. Profile sheet is visible and interactive. |
| Dismiss profile sheet | Slide down. Globe is full-screen and interactive. |

**Sheet nesting model:** Profile sheet is the primary slot. Trip detail sheet is nested inside the primary slot. Both sheets must remain functional simultaneously — this is the INFRA-02 spike validation. No cascading dismissal: dismissing trip detail must NOT auto-dismiss profile sheet.

Source: INFRA-02, GLOBE-03, GLOBE-05, DSYS-04 (REQUIREMENTS.md); Phase 1 success criteria item 3 (ROADMAP.md)

## AppFont Implementation Contract

The `AppFont` enum is the canonical font access point for all phases. Established in Phase 1, consumed by Phases 2–4.

```swift
// Contract — executor must implement this shape exactly
// Two weights only: Regular (400) and Semibold (600)
// Four sizes only: 28pt, 20pt, 16pt, 13pt
enum AppFont {
    static func title() -> Font       // 28pt Playfair Display Semibold (600)
    static func subheading() -> Font  // 20pt Playfair Display Regular (400)
    static func body() -> Font        // 16pt Inter Regular (400)
    static func caption() -> Font     // 13pt Inter Regular (400)
    static func buttonLabel() -> Font // 16pt Inter Semibold (600) — reuses Body size slot
}
```

Both Playfair Display and Inter must be registered in `Info.plist` as `UIAppFonts` entries. Both fonts must be registered at both Regular and Semibold variants. Executor must verify `UIFont(name:size:)` returns non-nil for both font families and both weights before the phase is declared complete.

Source: D-08 (CONTEXT.md), DSYS-01, DSYS-02 (REQUIREMENTS.md). `largeTitle()` (34pt) removed from the contract — Title (28pt) is the top of the scale.
