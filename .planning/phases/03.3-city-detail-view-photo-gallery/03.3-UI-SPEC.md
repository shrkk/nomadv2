## Color

All colors use the established `Color.Nomad` namespace (Phase 03.2 redesign). No new color
values are introduced in Phase 3.3.

| Role | Hex | SwiftUI Name | Usage |
|------|-----|-------------|-------|
| Dominant (60%) | `#0A0A0A` | `Nomad.panelBlack` | Sheet background; photo placeholder; card backgrounds |
| Globe BG (deep) | `#0A0A14` | `Nomad.globeBackground` | Underlying globe scene ŌĆö not used in sheets |
| Primary text | `#FFFFFF` | `Nomad.textPrimary` | All primary headings, values, city names |
| Secondary text | `#FFFFFF 60%` | `Nomad.textSecondary` | Sublabels, dates, secondary copy |
| Accent (10%) | `#FFFFFF` | `Nomad.accent` | See reserved list below |
| Destructive | `#D94F3D` | `Nomad.destructive` | Not used in Phase 3.3 (no destructive actions) |

**Accent (`Nomad.accent`) reserved for in Phase 3.3:**
1. Selected city thumbnail card border (2pt white stroke on image area when selected)
2. Selected city strip indicator dot (4pt white circle below selected card label)
3. Stats pill stat values (numeric foreground on all three stats)
4. "Open Settings" link in photo permission denied state

**Card surface:** All inner cards (trip log cards, stats pill) use `Color.black.opacity(0.35)` with
`Color.white.opacity(0.15)` hairline stroke via `innerCardSurface()` modifier from `PanelGradient.swift`.

**Photo placeholder:** Use `Color.Nomad.panelBlack` at full opacity as the loading placeholder
for the multi-photo carousel slides before images resolve.

**Glass surface:** All sheet backgrounds use `.panelGlassSurface()` / `.panelGradient()` from
`PanelGradient.swift` ŌĆö 4-layer background (panelBlack + ultraThinMaterial + dark overlay + vignette).

Source: `AppColors.swift`, `PanelGradient.swift` (Phase 03.2 redesign); 03.3-CONTEXT.md

## New Visual Specifications (Phase 3.3)

### City-Focused View Layout

When a city is selected (tapping a city thumbnail card transitions into city focus mode):

| Property | Specification |
|----------|--------------|
| Sheet container | Same `CountryDetailSheet` ŌĆö no new sheet. `.large` detent, `.panelGradient()` background. |
| Country header | Persistent at top-left. Shows country name (e.g. "United States") at all times. Back chevron dismisses the sheet. Header does NOT change when navigating cities. |
| City strip | Remains visible immediately below the country header. Selection indicator (white dot + white border) tracks the active city. |
| Content below strip | Replaced by city-specific content: multi-photo carousel ŌåÆ stats pill ŌåÆ trip logs. Layout is identical structure to country detail (carousel ŌåÆ stats ŌåÆ logs) but scoped to the selected city. |
| Transition animation | `.animation(.spring(response: 0.35, dampingFraction: 0.85), value: selectedCityIndex)` ŌĆö the content area crossfades/slides as `selectedCityIndex` changes. Use `withAnimation` block when setting `selectedCityIndex`. |
| Vertical scroll | `ScrollView(.vertical)` wrapping `VStack` ŌĆö city strip + city content scroll together as one unit. City strip is NOT pinned/sticky. Simpler non-sticky layout used. |

Source: 03.3-CONTEXT.md D-01, D-04, D-07; specifics (user quote about United States header)

### Full-Page City Swipe (Outer Swipe Navigation)

Swiping the entire sheet content left/right navigates between cities in the country.

| Property | Specification |
|----------|--------------|
| Gesture mechanism | Outer `TabView(selection: $viewModel.selectedCityIndex)` wrapping the entire content area below the country header (city strip included). `.tabViewStyle(.page(indexDisplayMode: .never))`. This makes city strip + photos + stats + logs all swipe as one unit per city. |
| Swipe unit | One full page = entire city content (city strip highlighting + photo carousel + stats + logs). Swiping left/right pages to next/previous city. |
| Sync with city strip | `selectedCityIndex` binding drives both the outer city TabView and the city strip's selected state. City strip highlighting automatically tracks swipe position via binding. |
| Page indicator | None separate ŌĆö the city strip itself serves as the city page indicator (selected city thumbnail highlighted with white border + dot). No additional dot indicator row is needed. |
| Transition | System TabView page swipe ŌĆö full-width horizontal scroll snap. |
| Swipe collision | The inner photo carousel (horizontal) and outer city swipe (horizontal) occupy different gesture zones. The outer swipe is triggered from edges or from vertical scroll gesture ŌĆö use `DragGesture` on outer container only if system TabView conflict arises. Default: rely on system TabView paging which handles nested scroll conflict gracefully in SwiftUI. |

Source: 03.3-CONTEXT.md D-08, D-09; specifics (magazine-swipe feel, city strip syncs)

### Multi-Photo Carousel (Inner, Per City)

The photo carousel within each city page shows ALL photos for that city.

| Property | Specification |
|----------|--------------|
| Implementation | Nested `TabView` (inner) with `.tabViewStyle(.page(indexDisplayMode: .never))`. Selection: local `@State var photoIndex: Int = 0` (not shared with selectedCityIndex). |
| Page width | Full screen width (edge to edge) |
| Carousel height | 320pt fixed |
| Corner radius | `RoundedRectangle(cornerRadius: 16)` clipping the entire carousel frame |
| Photo count | All PHAssets for the selected city's date range + GPS bounding box. No artificial limit. |
| Photo sort order | Chronological ascending (earliest to latest by `creationDate`) |
| Photo content mode | `.scaledToFill`, clipped to 320pt height. |
| Photo placeholder (loading) | `Color.Nomad.panelBlack` at 320pt height ŌĆö no skeleton animation needed |
| Photo placeholder (no photos) | `Color.Nomad.panelBlack` at 320pt height + centered caption: "No photos for {CityName}." (`AppFont.caption()`, `Nomad.textSecondary`) |
| Page indicator (within city photos) | Dot indicator row: only shown when city has 2+ photos. `HStack` of 8pt circles: filled `Nomad.accent` for current photo, `Nomad.textSecondary` (40% opacity) for others. Centered horizontally. Positioned 8pt below carousel frame, overlaid at bottom of carousel via `.overlay(alignment: .bottom)` with 8pt bottom inset. |
| Temperature notch | Per-photo overlay: same `TemperatureNotchPill` positioned 12pt from top of ZStack. Temperature is per-cluster (one value per city ŌĆö not per photo). |
| Photo tap behavior | Disabled ŌĆö no tap action, no full-screen viewer, no zoom. (D-05) |
| Photo loading strategy | Lazy: load photos for visible city only. Preload the adjacent city (prev + next) in background. PHImageManager `requestImage` with `.opportunistic` deliveryMode ŌĆö display degraded thumbnail immediately, replace with high-quality on completion. Guard against degraded-frame display using `PHImageResultIsDegradedKey` check for final render only (allow degraded for placeholder). |
| Memory management | Limit concurrent high-quality loads to 3 cities at a time (visible + immediate neighbors). Evict distant cities' full-res images from `[UUID: [UIImage]]` dictionary when `selectedCityIndex` changes by more than 2 steps. |

Source: 03.3-CONTEXT.md D-02, D-05, D-06 (Claude's discretion on loading strategy)

## Interaction Contract

| Interaction | Behavior |
|-------------|---------|
| Tap city thumbnail in strip | `selectedCityIndex` updates. Outer city TabView animates to that city page. Photo carousel resets to first photo (photoIndex = 0). Spring: response 0.35, damping 0.85. |
| Swipe entire sheet content left/right | Outer city TabView pages to next/previous city. `selectedCityIndex` updates. City strip indicator dot + border follow. |
| Swipe photo carousel left/right (inner) | Inner photo TabView pages to next/previous photo within current city. Does NOT affect `selectedCityIndex`. Page indicator dots update. |
| Tap photo | No action ŌĆö gesture disabled on photo area. |
| Tap trip log card | `TripDetailSheet` slides up over CountryDetailSheet (nested per INFRA-02). Spring: damping 0.85, response 0.4. |
| Tap back chevron (top-left) | Sheet dismisses. Globe returns to full-screen. |
| Swipe sheet down | System drag-to-dismiss. Same result as back chevron. |
| Swipe down TripDetailSheet | TripDetailSheet dismisses. CountryDetailSheet remains. |
| Temperature notch | Display only ŌĆö no tap action. |
| Stats pill | Display only ŌĆö no tap action. |

## State Summary

| State | Trigger | Visual |
|-------|---------|--------|
| Sheet loading | On appear, before data resolves | `ProgressView()` centered below country header; city strip skeleton (3 ├Ś `Nomad.panelBlack` rounded rects 64├Ś80pt) |
| Sheet loaded | City clusters resolved | City strip populated; first city's carousel visible |
| City selected / swiped to | User taps city card or swipes to city page | White border on city card; white dot below label; carousel shows that city's photos |
| Photo loading (per city) | PHAsset fetch in progress | `Nomad.panelBlack` placeholder at 320pt height for unfilled slots |
| Photo loaded (degraded) | `PHImageManager` returns degraded frame | Show degraded thumbnail immediately (low-res preview) |
| Photo loaded (final) | `PHImageManager` returns high-quality, `isDegraded = false` | Replace with full-resolution image |
| No photos for city | 0 PHAssets matched for date range | Carousel shows `Nomad.panelBlack` 320pt + "No photos for {CityName}." caption |
| Temperature loading | WeatherKit fetch in progress | "ŌĆö┬░C" in notch pill |
| Temperature unavailable | WeatherKit error or no data | Pill hidden entirely |
| No trip logs for city | 0 trips in cluster | "No trips logged for {CityName}." in logs section |
| No cities for country | 0 clusters | "No visits yet." empty state below header |
| Photo permission denied | `PHPhotoLibrary.authorizationStatus` is `.denied` or `.restricted` | Carousel placeholder + "Allow photo access in Settings." + "Open Settings" link |
