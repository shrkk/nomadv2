# Phase 03.2 Plan 03: Live Activity Widget Summary

**One-liner:** ActivityKit Live Activity with Dynamic Island compact/expanded + Lock Screen banner showing active trip distance, elapsed time, and city-level location, wired into the existing trip lifecycle via LocationManager.

---

## What Was Built

### TripActivityAttributes.swift (NomadLiveActivity/)
Shared `ActivityAttributes` struct containing `ContentState` with exactly four fields:
- `distanceKm: Double` — kilometers covered
- `elapsedSeconds: Int` — seconds since trip start
- `locationName: String` — city-level reverse-geocoded name, max 30 chars
- `isRecording: Bool` — true while active, false on end

Security note: no trip ID, user ID, or precise GPS coordinates in ContentState (T-03.2-03 mitigated).

### TripLiveActivity.swift (NomadLiveActivity/)
`ActivityConfiguration<TripActivityAttributes>` with three layouts:

**Compact Dynamic Island (pill):**
- Leading: pulsing white dot (8pt, easeInOut 0.8s, scaleEffect 1.5) + elapsed "1h 23m"
- Trailing: distance "4.2 mi" (locale-aware via MeasurementFormatter)

**Expanded Dynamic Island (long-press):**
- Row 1: pulsing dot + "Recording" label + spacer + elapsed (compact format)
- Row 2: distance stat block | white divider | elapsed stat block (detailed "1h 23m 14s")
- Row 3: location.fill icon + city name (max 30 chars, white 80% opacity)

**Lock Screen Banner:**
- Leading VStack: dot + "Nomad" app label (white 70%) / city name (white, lineLimit 1)
- Trailing VStack: distance (white, "Starting…" when <0.1 km) / elapsed compact (white 70%)

### NomadLiveActivityBundle.swift (NomadLiveActivity/)
`@main WidgetBundle` entry point registering `TripLiveActivity()`.

### LocationManager.swift (extended)
New properties: `accumulatedDistanceMeters`, `recordingStartDate`, `lastLocation`, `liveActivityUpdateTimer`, `lastGeocodedLocationName`, `lastGeocodeTime`

New methods:
- `startLiveActivity()` — requests Activity<TripActivityAttributes> with initial state
- `endLiveActivity()` async — ends all active activities with final stats, `.immediate` dismissal
- `updateLiveActivity()` private — pushes current state to active activities (called by timer)

Changes to existing methods:
- `startRecording()` — resets distance/location state, sets `recordingStartDate`, starts 30s timer
- `stopRecording()` — invalidates timer before cleanup, resets all distance state
- `saveRoutePoint()` — accumulates distance from consecutive GPS fixes, throttled reverse geocode

### GlobeView.swift (wired)
- `import ActivityKit` added
- Both trip start paths (JourneyPill `onStartTrip` + ProfileSheet `onStartTrip`) end stale activities then call `locationManager.startLiveActivity()`
- `saveTrip()` calls `await locationManager.endLiveActivity()` after `stopRecording()`
- `discardTrip()` calls `endLiveActivity()` in a `Task` after `stopRecording()`
