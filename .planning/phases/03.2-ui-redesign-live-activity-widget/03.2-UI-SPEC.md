# Phase 03.2 — UI Design Contract

> Visual and interaction contract for the full design system replacement (warm amber/cream → black/white glassmorphic) and the new ActivityKit Live Activity widget (Dynamic Island + Lock Screen banner).
> Generated by gsd-ui-researcher. All decisions sourced from 03.2-CONTEXT.md unless marked [default].

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (native SwiftUI + UIKit) |
| Preset | not applicable |
| Component library | SwiftUI native + UIVisualEffectView |
| Icon library | SF Symbols (existing — unchanged) |
| Font | Inter (all weights — Playfair Display removed) |

**iOS target:** iOS 16.2+ (ActivityKit minimum). Verify deployment target before implementing Live Activity.

## Spacing Scale

All padding and layout values are multiples of 4pt. This is a native iOS SwiftUI project; "px" maps 1:1 to SwiftUI points.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4pt | Icon-to-label gaps, hairline offsets |
| sm | 8pt | Intra-element spacing (e.g. dot-to-text in pills) |
| md | 16pt | Default horizontal/vertical padding for all panels and sheets |
| lg | 24pt | Section separation within sheets |
| xl | 32pt | Major section breaks |
| 2xl | 48pt | Bottom scroll clearance (sheet safe area) |
| 3xl | 64pt | Bottom clearance for sheets extending to safe area |

**Exceptions:**
- Touch targets: minimum 44×44pt for all tappable elements (SF HIG requirement)
- Panel drag handle: 36×4pt capsule, 8pt top padding — fixed size, not derived from scale
- Dynamic Island compact pill: system-controlled outer bounds — do not pad beyond system constraints
- RecordingPill min-height: 44pt (existing touch target preserved)

## Typography

All call sites use `AppFont.*` API — only the font name strings inside `AppFont.swift` change.
Playfair Display is removed. Inter is already bundled; no new font assets needed.

| Role | AppFont API | Font | Size | Weight | Line Height |
|------|-------------|------|------|--------|-------------|
| Title | `AppFont.title()` | Inter-Bold | 28pt | 700 | 1.2 |
| Subheading | `AppFont.subheading()` | Inter-SemiBold | 20pt | 600 | 1.3 |
| Body | `AppFont.body()` | Inter-Regular | 16pt | 400 | 1.5 |
| Caption | `AppFont.caption()` | Inter-Regular | 13pt | 400 | 1.4 |
| Button Label | `AppFont.buttonLabel()` | Inter-SemiBold | 16pt | 600 | 1.0 (single line) |

**Validation assertions in `AppFont.swift` DEBUG block:** Update from `PlayfairDisplay-Regular` / `PlayfairDisplay-SemiBold` to `Inter-Bold` / `Inter-SemiBold`. Inter-Regular and Inter-SemiBold assertions already exist and stay unchanged.

**Live Activity typography** (WidgetKit/ActivityKit text — not AppFont, uses system font API):
- Compact pill label: `.system(size: 13, weight: .semibold, design: .rounded)` — system rounded for legibility at small size
- Expanded Island primary values: `.system(size: 16, weight: .semibold)`
- Expanded Island labels: `.system(size: 12, weight: .regular)`
- Lock Screen banner primary: `.system(size: 15, weight: .semibold)`
- Lock Screen banner secondary: `.system(size: 13, weight: .regular)`

## Color

### Core Token Replacement

Replace `AppColors.swift` (`Color.Nomad` extension) entirely. Old tokens `cream`, `warmCard`, `amber` are removed. New token set:

| Token | Value | Hex | Usage |
|-------|-------|-----|-------|
| `panelBlack` | near-black | `#0A0A0A` | Primary panel/sheet backgrounds — dominant surface (60%) |
| `surfaceGlass` | computed | black at 30% opacity + `.ultraThinMaterial` blur | Card surfaces, inner containers — secondary (30%) |
| `textPrimary` | pure white | `#FFFFFF` | All primary text — body, headings, button labels |
| `textSecondary` | white 60% opacity | `#FFFFFF` at 0.60 alpha | Secondary labels, captions, subtitles, dividers |
| `accent` | pure white | `#FFFFFF` | Accent (10%) — reserved elements listed below |
| `destructive` | unchanged | `#D94F3D` | Destructive actions only — Stop Trip confirmation, discard |
| `globeBackground` | unchanged | `#0A0A14` | RealityKit/ARView scene background — never used in SwiftUI panels |

**Accent (`#FFFFFF`) reserved for these specific elements only:**
- Active trip stat values in the RecordingPill (elapsed time text)
- Button border hairlines (1pt stroke at 20% white opacity)
- "Stop Trip" button label text
- Key stat values in StatsRow (steps, distance, duration, stops values — not their labels)
- SF Symbol tints on the `plus.circle.fill` icon in JourneyPill
- The pulsing recording dot in RecordingPill and Live Activity
- Country polygon highlight fill (white at 40% opacity — see Globe section)
- `Open Settings` link in CityPhotoCarousel photo-denied state

**Do NOT use accent for:** body text, captions, non-interactive decorative elements, dividers, or background fills.

**60/30/10 split:**
- 60% dominant: `panelBlack` (#0A0A0A) — all sheet/panel backgrounds
- 30% secondary: `surfaceGlass` — stat row backgrounds, thumbnail card backgrounds, inner containers
- 10% accent: pure white (`#FFFFFF`) — reserved elements listed above only

**Color.Nomad implementation contract:**

```swift
extension Color {
    enum Nomad {
        static let globeBackground  = Color(red: 0.039, green: 0.039, blue: 0.078) // #0A0A14 — unchanged
        static let panelBlack       = Color(red: 0.039, green: 0.039, blue: 0.039) // #0A0A0A
        static let textPrimary      = Color.white
        static let textSecondary    = Color.white.opacity(0.60)
        static let accent           = Color.white
        static let destructive      = Color(red: 0.851, green: 0.310, blue: 0.239) // #D94F3D — unchanged
        // surfaceGlass: use .ultraThinMaterial + panelBlack overlay at call site
        // (not a flat Color value — requires material modifier)
    }
}
```

## Glassmorphic Surface Contract

Every bottom sheet panel and floating pill uses this exact surface recipe. Executor must apply uniformly.

### Panel Surface (ProfileSheet, TripDetailSheet, CountryDetailSheet)

```
Background stack (bottom to top):
  1. Color.Nomad.panelBlack          — solid opaque base
  2. .ultraThinMaterial               — SwiftUI material blur (UIBlurEffect style: .systemUltraThinMaterialDark)
  3. Color.black.opacity(0.35)        — darkness reinforcement over blur
  4. RoundedRectangle stroke           — 1pt, Color.white.opacity(0.20), top corners only
```

Corner radius: 20pt top corners (`UnevenRoundedRectangle(topLeadingRadius: 20, topTrailingRadius: 20, bottomLeadingRadius: 0, bottomTrailingRadius: 0)`) — existing shape preserved.

Drag handle pill: 36×4pt capsule, `Color.white.opacity(0.30)`, centered, 8pt top padding.

### Floating Pill Surface (RecordingPill, JourneyPill)

```
Background stack:
  1. Capsule filled with .ultraThinMaterial (dark)
  2. Color.Nomad.panelBlack.opacity(0.80) overlay
  3. Capsule stroke: 1pt, Color.white.opacity(0.20)
```

Shadow: `.shadow(color: .black.opacity(0.40), radius: 12, x: 0, y: 6)` — slightly deeper than current.

### Inner Card Surface (StatsRow, TripLogCard, CityThumbnailCard, StatsPillRow)

```
Background:
  1. Color.black.opacity(0.35) fill
  2. RoundedRectangle stroke: 1pt, Color.white.opacity(0.15)
Corner radius: 12pt
```

### PanelGradient Replacement

`PanelGradient.swift` is replaced. The new `PanelGlassSurface` modifier applies the panel surface contract above. No amber corner bleeding. Optional edge darkening:

```
Edge darkening (Claude's discretion — apply as subtle vignette):
  RadialGradient(
    colors: [.clear, Color.black.opacity(0.20)],
    center: .center,
    startRadius: UIScreen.main.bounds.width * 0.4,
    endRadius: UIScreen.main.bounds.width * 0.9
  )
```

The grain noise overlay (`Image("grain-noise")`) is removed entirely. True glass surfaces do not use grain.

### Button Style Contract

All interactive buttons across the redesigned surfaces:

```
Shape: Capsule (pill-shaped) for standalone buttons; RoundedRectangle(cornerRadius: 8) for inline buttons
Fill: Color.black.opacity(0.35) + .ultraThinMaterial
Border: 1pt stroke, Color.white.opacity(0.20)
Text: Color.Nomad.textPrimary (white), AppFont.buttonLabel() (Inter-SemiBold 16pt)
Icon tint: Color.Nomad.textPrimary
Press state: scaleEffect(0.96) + opacity(0.85), animation .easeInOut(duration: 0.12)
Minimum touch target: 44×44pt
```

**Destructive button variant** (Stop Trip, Discard):
```
Text/Icon color: Color.Nomad.destructive (#D94F3D)
Border: 1pt stroke, Color.Nomad.destructive.opacity(0.40)
Fill: Color.black.opacity(0.35)
```

## Pulsing Recording Dot — Animation Spec

Used in RecordingPill (in-app) and Live Activity (Dynamic Island + Lock Screen).

```swift
// Fill: Color.white (in-app) / white (WidgetKit)
// Size: 8pt diameter
// Animation:
Circle()
    .fill(Color.white)
    .frame(width: 8, height: 8)
    .scaleEffect(isPulsing ? 1.5 : 1.0)
    .opacity(isPulsing ? 0.5 : 1.0)
    .animation(
        .easeInOut(duration: 0.8).repeatForever(autoreverses: true),
        value: isPulsing
    )
    .onAppear { isPulsing = true }
```

Pulse rhythm: 0.8s half-period (scale 1.0→1.5→1.0 in 1.6s full cycle). This is tighter than the current 1.0s half-period — matches heartbeat cadence as specified.

**In-app PulseAnimation ViewModifier:** Update existing `PulseAnimation` modifier to use duration: 0.8 (from 1.0) and scaleEffect max: 1.5 (from 1.4).

## Live Activity — ActivityKit Contract

### Architecture

```
TripActivityAttributes : ActivityAttributes
    struct ContentState : Codable, Hashable
        var distanceKm: Double        // kilometers covered, 1 decimal precision
        var elapsedSeconds: Int       // seconds since trip start
        var locationName: String      // reverse-geocoded, max 30 chars, truncated with ellipsis
        var isRecording: Bool         // always true while active; false triggers end animation
```

`TripActivityAttributes` holds no static properties (all data is dynamic, lives in ContentState).

**Activity lifecycle:**
- Start: when user starts a trip (manual "+" or auto-detect prompt accepted), before `locationManager.startRecording(tripId:)` returns
- Update: `LocationManager` calls `Activity<TripActivityAttributes>.activities.first?.update(using: newState)` — throttled, see update frequency below
- End: when `TripService.finalizeTrip` completes successfully. End with `.immediate` dismissal policy — the widget disappears without lingering
- Only one `TripActivityAttributes` activity is active at a time. Start sequence must call `.end` on any existing activity before `.request`

**Update frequency:**
- Distance/elapsed: update every 30 seconds from `LocationManager` background task
- Location name: update at most every 60 seconds (CLGeocoder throttle). Cache last geocoded result; skip update if location name is unchanged
- Do not update if app is in foreground and Live Activity is not visible — unnecessary battery drain

**Stale state:** If app is suspended for >5 minutes and no updates arrive, `isRecording` in ContentState will remain `true` — this is acceptable for v1 (local-only, no push tokens)

### Dynamic Island — Compact State

Displayed in the Dynamic Island pill during an active trip (not expanded).

```
Layout: HStack, system-constrained width (~120pt max)
  Leading (left of pill notch):
    HStack(spacing: 4):
      Circle (pulsing dot), 8pt, white
      Text(elapsedFormatted), .system(size: 13, weight: .semibold, design: .rounded), white
  Trailing (right of pill notch):
    Text(distanceFormatted), .system(size: 13, weight: .semibold, design: .rounded), white
```

`elapsedFormatted`: "1h 23m" (hours + minutes only, drop seconds in compact)
`distanceFormatted`: "4.2 km" or "4.2 mi" based on locale

**Compact leading region:** pulsing dot + elapsed time
**Compact trailing region:** distance only

### Dynamic Island — Expanded State

Long-press to expand. Fills the full Dynamic Island expanded area.

```
Layout: VStack(spacing: 8), padding(.horizontal, 16).padding(.vertical, 12)
Background: black (system-provided in Dynamic Island context)

Row 1 — Recording header:
  HStack(spacing: 8):
    Circle (pulsing dot), 8pt, white
    Text("Recording"), .system(size: 16, weight: .semibold), white
    Spacer()
    Text(elapsedFormatted), .system(size: 16, weight: .semibold), white
    // elapsedFormatted here shows hours + minutes only

Row 2 — Stats:
  HStack(spacing: 0):
    statBlock(value: distanceFormatted, label: "Distance")
    Divider().frame(height: 28).overlay(Color.white.opacity(0.25))
    statBlock(value: elapsedFormatted, label: "Elapsed")
    // elapsedFormatted here shows hours + minutes + seconds e.g. "1h 23m 14s"

Row 3 — Location:
  HStack(spacing: 6):
    Image(systemName: "location.fill"), 11pt, white 60% opacity
    Text(locationName), .system(size: 13, weight: .regular), white 80% opacity
    // Max 30 chars, truncated with trailing "…" if longer
    Spacer()
```

`statBlock(value:label:)`:
```
VStack(spacing: 2):
  Text(value), .system(size: 16, weight: .semibold), white
  Text(label), .system(size: 11, weight: .regular), white 60% opacity
.frame(maxWidth: .infinity)
```

**Expanded state dimensions (system-provided canvas, approximate):**
- Width: ~350pt (full Dynamic Island expanded width)
- Height: ~120pt (system-allocated)
- Do not hardcode dimensions — use `.frame(maxWidth: .infinity)` and let the system constrain

## Interaction States

### Tab / Tap Feedback (all buttons)

```
Normal:    opacity 1.0, scale 1.0
Pressed:   opacity 0.85, scale 0.96
Animation: .easeInOut(duration: 0.12) on press, .spring(response: 0.3, dampingFraction: 0.7) on release
```

Use `.buttonStyle(GlassButtonStyle())` custom ButtonStyle conformance for consistency.

### CityThumbnailCard — Selected State

```
Selected:   border 1pt white full opacity, scale 1.02
Unselected: border 1pt white 15% opacity, scale 1.0
Transition: .animation(.spring(response: 0.35, dampingFraction: 0.8), value: isSelected)
```

### Loading / Skeleton State

All loading states use `ProgressView()` with `.tint(Color.Nomad.textPrimary)` (white spinner). No skeleton shimmer — the dark surfaces make white spinners legible without additional treatment.
