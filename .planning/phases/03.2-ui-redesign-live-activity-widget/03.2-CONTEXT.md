# Phase 3.2: UI Redesign & Live Activity Widget — Context

**Gathered:** 2026-04-08
**Status:** Ready for planning
**Source:** Interactive discussion with user

<domain>
## Phase Boundary

Two parallel deliverables:

1. **Full visual redesign** — Replace the warm amber/cream design system with a true black/white minimalist aesthetic using frosted dark glass surfaces across every screen the user touches: globe home chrome, all bottom sheets, active trip UI, and all interactive buttons.

2. **Live Activity widget** — A new ActivityKit-powered widget showing active trip stats in the Dynamic Island (compact + expanded) and a Lock Screen banner — both greenfield (no existing Live Activity code in the project).

This phase is inserted between Phase 3 (Core User Journey) and Phase 4 (Passport), so all redesigned views must continue to work with the Phase 2/3 data layer unchanged.

</domain>

<decisions>
## Implementation Decisions

### D-01: Color Palette — Full Replacement
The warm amber (#E8A44A) + cream (#FAF8F4 / #F5F0E8) palette is **fully replaced**. All new tokens:

| Token | Value | Usage |
|-------|-------|-------|
| `panelBlack` | `#0A0A0A` (near-black) | Primary panel/sheet backgrounds |
| `surfaceGlass` | Black 30–40% opacity + blur | Card surfaces, inner containers |
| `textPrimary` | `#FFFFFF` (pure white) | All primary text |
| `textSecondary` | White 60% opacity | Secondary labels, captions |
| `accent` | `#FFFFFF` (pure white) | Active states, highlights, key stats, button borders |
| `destructive` | Unchanged `#D94F3D` | Destructive actions only |
| `globeBackground` | Unchanged `#0A0A14` | Globe scene background only |

The old `cream`, `warmCard`, and `amber` tokens in `AppColors.swift` are replaced. Claude's discretion on exact opacity values for glass surfaces.

**Scope:** Replace `Nomad/DesignSystem/AppColors.swift` entirely. All call sites (sheets, components) must be updated to use new tokens.

### D-02: Glassmorphic Surface Style
- **Dark frosted glass** — UIBlurEffect / SwiftUI `.ultraThinMaterial` with a dark underlying layer
- Implementation: `.background(.ultraThinMaterial)` over a black base, or `UIVisualEffectView(effect: UIBlurEffect(style: .systemUltraThinMaterialDark))`
- Glass surfaces have a subtle 1pt white border at ~20% opacity (hairline edge to catch light)
- Panel gradient (`PanelGradient.swift`) is **replaced** — no more amber corner bleeding; new style is a very subtle radial darkening at panel edges (optional, Claude's discretion)
- All bottom sheet panels use this style: `ProfileSheet`, `TripDetailSheet`, `CountryDetailSheet`

### D-03: Typography — Drop Playfair Display, Go All-Inter
Replace `AppFont.swift` with an all-Inter scale:

| Function | Font | Size |
|----------|------|------|
| `title()` | Inter-Bold | 28pt |
| `subheading()` | Inter-SemiBold | 20pt |
| `body()` | Inter-Regular | 16pt |
| `caption()` | Inter-Regular | 13pt |
| `buttonLabel()` | Inter-SemiBold | 16pt |

Playfair Display is **removed** — Inter is already bundled, no new font assets needed.

**Note:** All call sites using `AppFont.*` continue to compile — only the underlying font changes, the API stays identical.

### D-04: Globe — Chrome Only, Not Sphere
The globe sphere, dark space background, and star field are **unchanged** (they already match the black/white aesthetic).

What changes:
- The bottom handle/drag indicator on the profile panel slides up over the globe — redesign to glass pill on dark background
- Any floating controls over the globe (active trip indicator, country highlight colors) — align to new palette
- Country polygon highlight color: consider switching from amber fill to **white fill at 40% opacity** for visited countries; Claude's discretion on exact value (must be readable against the satellite imagery)

### D-05: Redesign Scope — All Screens
Every user-facing surface gets the new treatment:
- **Globe home chrome** — panel handle, floating active-trip indicator, any controls overlaid on globe
- **All bottom sheets** — `ProfileSheet`, `TripDetailSheet`, `CountryDetailSheet`, `CityPhotoCarousel`
- **Active trip UI** — the recording indicator and Stop/Name Trip controls
- **All buttons** — frosted glass style: dark blur background + white hairline border + white text/icon

### D-06: Live Activity — Dynamic Island + Lock Screen
New feature, greenfield. Uses ActivityKit framework.

**Dynamic Island — Compact state** (minimal pill):
- Left side: pulsing recording dot (white, animated)
- Right side: elapsed time (e.g. "1h 23m")

**Dynamic Island — Expanded state** (long-press / auto-expand):
- Row 1: pulsing dot + "Recording" label
- Row 2: Distance covered (e.g. "4.2 mi"), Elapsed time (e.g. "1h 23m")
- Row 3: Current location name (geocoded, e.g. "Mission District, SF")

**Lock Screen Live Activity banner:**
- Same data as expanded Dynamic Island
- Layout: leading = pulsing dot + location name; trailing = distance + elapsed time
- App name "Nomad" as label

**Data sources:**
- Distance: computed from accumulated GPS route points in LocationManager
- Elapsed time: computed from trip start timestamp
- Current location: reverse geocode of most recent GPS point (CLGeocoder, throttled — update max every 60s)
- Recording dot: always animated while activity is live

**Lifecycle:**
- Activity starts when user starts a trip (manual or auto-detect prompt accepted)
- Activity ends when user stops and names the trip
- Single active activity at a time (previous activity ended before new one starts)

**Implementation notes (Claude's discretion):**
- Stale state handling if app is suspended
- ActivityKit `ActivityAttributes` and `ContentState` struct design
- Push token for remote updates (not required for v1 — local updates only)

### Claude's Discretion
- Exact blur radius and opacity values for glass surfaces
- Country polygon highlight color in new palette (white fill at ~40% opacity recommended)
- Subtle panel edge darkening in new `PanelGradient` replacement
- CLGeocoder throttling interval for Live Activity location updates
- ActivityKit ContentState update frequency (local, no push required for v1)
- Empty states for Live Activity (no GPS data yet, no location name available)
- Whether to animate the glass button press state (scale/opacity on tap)

<specifics>
## Specific Ideas

- True monochrome: black/white/glass with NO amber anywhere — not even as a vestigial accent. Pure white is the sole highlight color.
- Globe sphere unchanged — the existing dark space aesthetic already fits the new palette naturally.
- Typography pivot is minimal in code impact: `AppFont.*` call sites stay identical; only the font name strings inside `AppFont.swift` change from Playfair → Inter.
- Dynamic Island recording dot should pulse like a heartbeat — not just a static dot. White fill, `withAnimation(.easeInOut(duration: 0.8).repeatForever(autoreverses: true))` scale pulse.
- Live Activity layout should feel like Apple's own Workout/Maps activities — information dense in compact, spacious in expanded.

</specifics>
