# Technology Stack

**Analysis Date:** 2026-04-10

## Languages

**Primary:**
- Swift 6.0 - iOS app (`Nomad` target, IPHONEOS_DEPLOYMENT_TARGET = 18.0)
- Swift 5.0 - Live Activity widget extension (`NomadLiveActivity` target, IPHONEOS_DEPLOYMENT_TARGET = 16.2)
- TypeScript 5.0.0 - Firebase Cloud Functions (Node.js 20)

**Secondary:**
- Plist/XML - Configuration and app metadata
- JSON - Configuration and data serialization

## Runtime

**Environment:**
- iOS 18.0 (main app)
- iOS 16.2 (Live Activity widget extension)
- Node.js 20 (Firebase Functions)

**Package Manager:**
- Swift Package Manager (SPM) - Used for iOS dependencies via XCRemoteSwiftPackageReference
- npm - Used for Firebase Functions (Node.js)
- Lockfile: `package-lock.json` present for functions

## Frameworks

**Core iOS:**
- SwiftUI - UI framework (imported across all UI targets)
- SwiftData - Local data persistence for GPS points and trip metadata (`RoutePoint`, `TripLocal` models)
- Combine - Async/reactive programming patterns
- Observation - @Observable pattern for state management (Swift 6)

**Location & Maps:**
- CoreLocation - GPS tracking, geofencing, visit monitoring (`CLLocationManager`, `CLVisit`, `CLCircularRegion`)
- MapKit - Route visualization, POI categorization via `MKLocalPointsOfInterestRequest` (PLACE-01)

**Device Features:**
- ActivityKit - Live Activity widget for active trips (`TripActivityAttributes`, `NomadLiveActivity` target)
- HealthKit - Step count integration for trip stats
- Photos - Photo library access for trip photo gallery matching
- UserNotifications - Local notifications for trip prompts and geofence alerts

**System Frameworks:**
- UIKit - Base iOS framework integration
- Foundation - Standard library
- RealityKit - 3D rendering for globe visualization
- AVFoundation - Not imported but potentially used by MapKit

**Backend & Cloud:**
- Firebase Core - Firebase initialization
- Firebase Authentication - Email/password and Google Sign-In
- Firebase Firestore - Realtime database for users, trips, route points
- Google Sign-In - OAuth2 authentication integration

**Testing/Compilation:**
- TypeScript 5.0.0 - Firebase Functions compilation
- Firebase Admin SDK (Node.js) - Server-side database and auth operations

## Key Dependencies

**Critical:**
- `firebase-admin` ^12.0.0 - Cloud Functions: user deletion cleanup, Firestore batch operations
- `firebase-functions` ^5.0.0 - Cloud Functions: HTTP triggers and auth hooks
- FirebaseAuth - iOS: User authentication state management
- FirebaseFirestore - iOS: Primary data store for user profiles, trips, route points
- GoogleSignIn - iOS: OAuth2 sign-in flow

**Infrastructure:**
- Google Sign-In iOS SDK - OAuth client for Google authentication
- Firebase iOS SDK - Includes Auth, Firestore, Core modules

**Swift 6 Compatibility:**
- Uses `@preconcurrency` import annotations for thread-safety with Firestore and MapKit
- @Observable and @MainActor patterns for concurrency isolation

## Configuration

**Environment:**
- Firebase project: `nomad-d92cb`
- `.firebaserc` specifies default Firebase project
- `GoogleService-Info.plist` embedded in app bundle (`Nomad` target) with CLIENT_ID, API_KEY, PROJECT_ID, STORAGE_BUCKET
- `firebase.json` configures Cloud Functions (runtime: nodejs20) and Firestore rules

**Build:**
- `Nomad.xcodeproj` - Main Xcode project with multiple targets
- Targets:
  - `Nomad` (main iOS app, Swift 6.0, iOS 18.0+)
  - `NomadLiveActivity` (widget extension, Swift 5.0, iOS 16.2+)
  - Live Activity extension embedded in main app bundle
- Build phases:
  - Copy fonts (PlayfairDisplay-Regular.otf, PlayfairDisplay-SemiBold.otf, Inter-Regular.ttf, Inter-SemiBold.ttf)
  - Link system frameworks (ActivityKit, WidgetKit, MapKit, CoreLocation)
  - Link Firebase frameworks via SPM

**Code Organization:**
- `functions/src/index.ts` - Single Firebase Cloud Function entry point
- `functions/lib/` - Compiled JavaScript output (generated by tsc)
- `functions/tsconfig.json` - TypeScript configuration with strict mode enabled
  - Compilation targets: ES2017
  - Modules: CommonJS
  - Output directory: lib/

## Platform Requirements

**Development:**
- Xcode 15+ (required for Swift 6.0)
- iOS 18.0 SDK (Xcode 16+)
- macOS (development environment)
- Node.js 20.x for Firebase Functions

**Production:**
- Deployment: Apple App Store (requires provisioning, code signing)
- Backend: Firebase Cloud (managed by Google)
- Database: Firestore (scales automatically)
- Authentication: Firebase Auth + Google OAuth
- Hosting: None (serverless backend via Firebase)

**Hardware/Permissions:**
- Device: iPhone (iOS 18.0+)
- Required permissions: Location (Always & When In Use), Photos, HealthKit
- Optional: UserNotifications (geofence alerts)
- Background modes: Location updates, geofencing
